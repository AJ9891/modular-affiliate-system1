# Voice Rules

Status: Operating Constitution
Scope: Product copy, AI responses, onboarding language, and UI system messaging.

## Mission
Voice must be clear, direct, and trustworthy while maintaining product authority.

## Tone Standards
1. Write in plain language first; precision over flourish.
2. Use confident, factual phrasing; avoid hype and vague claims.
3. Keep sentences short where operational clarity matters.
4. Respect user context; never blame the user.

## Structural Rules
1. Start with outcome, then details.
2. Use actionable verbs and explicit nouns.
3. Avoid layered jargon when one specific term exists.
4. Prefer scannable formatting for procedural content.

## Prohibited Patterns
1. No fabricated urgency ("act now", "limited" without evidence).
2. No unexplained absolutes ("always", "never") unless policy-backed.
3. No anthropomorphic claims that imply false AI capability.
4. No internally inconsistent naming for product concepts.

## UI Microcopy Rules
1. Buttons must describe action result, not only action type.
2. Errors must include what failed, why (if known), and next step.
3. Empty states must suggest first useful action.
4. Success states must confirm completion and optional next action.

## Review Rules
1. High-visibility copy changes require one human reviewer.
2. If voice conflicts with legal/compliance language, legal/compliance wins.
3. Repeated user confusion on wording triggers rule refinement in this file.
